# Director's Visual Design（导演视觉设定）

## 定位

本技能是导演将**个人创意决策**注入 AI 影视制作流水线的环节。

Art Direction（美术方向）定义的是全局视觉风格——色彩、光影、质感等"所有画面都必须遵守的规则"。而 Director's Visual Design 定义的是**具体元素**的视觉形象——"这个角色长什么样"、"这个场景是什么氛围"、"这个道具什么造型"。

在真实影视制作中，这对应导演与概念设计师一起做的 Visual Development / Lookdev 阶段——导演翻着参考图说"我想要这种感觉"，概念设计师据此出图。

**核心价值**：解决"原文外貌描写稀缺 → 模型补全趋同 → 角色撞脸"的问题。通过导演主动提供视觉偏好（文字或参考图），将创意决策从模型补全提升为人工定义。

## 优先级规则

下游环节（step3 圣经编写、step4 资产生成）在确定视觉形象时，严格按以下优先级：

```
导演视觉设定（本文件） > 小说原文描写 > 模型自主补全
```

- 导演设定中**有明确描述的**：以导演设定为准，原文描写作为补充参考
- 导演设定中**留空的**：从小说原文中提取，原文不足时由模型补全（但需标注"模型补全"）
- 导演设定中**明确标注"不限"的**：由模型自由发挥，但需确保与其他已定义的元素有足够区分度

## 输入

- `production/step1-extraction/characters.md` — 角色清单
- `production/step1-extraction/locations.md` — 场景清单
- `production/step1-extraction/props.md` — 道具清单
- `production/step2-visual-design/art_direction.md` — 全局视觉风格
- 导演的创意意图（通过对话交流获取）

## 输出

一份 Markdown 文件，存放在 `production/step2-visual-design/` 下：


## 各板块编写规范

### 〇、进度追踪

> 本板块是整个文件的"控制面板"，记录所有待处理元素的当前状态。**每次会话的第一件事是读这张表，最后一件事是更新这张表。**

```markdown
## 〇、进度追踪

> 最后更新：2026-03-23 Session 2（完成 5 个角色、2 个场景）

| 类别 | 元素名 | 重要度 | 状态 | 备注 |
|------|--------|--------|------|------|
| 角色 | 宋小仙 | ★★★★★ | ✅ 已完成 | |
| 角色 | 朱围庸 | ★★★★★ | ✅ 已完成 | |
| 角色 | 老吴   | ★★★★☆ | 🔄 进行中 | 选角偏好已定，阶段划分待确认 |
| 角色 | 黄毛   | ★★★☆☆ | ⬜ 待处理 | |
| 场景 | 石岭关 | ★★★★★ | ⬜ 待处理 | |
| 道具 | 追魂瓶 | ★★★★☆ | ⏭️ 跳过 | 导演明确无意见，委托 AI |
```

**状态定义**：

| 状态 | 含义 |
|------|------|
| ⬜ 待处理 | 尚未与导演讨论 |
| 🔄 进行中 | 已开始讨论，但尚有字段未确认（备注列写明哪些未完成） |
| ✅ 已完成 | 导演已确认所有字段 |
| ⏭️ 跳过 | 导演明确表示无意见，全权委托 AI |

**编写要点**：
- **"最后更新"行**必须写明日期和本次会话的概要（完成了什么），方便下次恢复时快速了解上下文
- **重要度**从 step1 提取结果中继承，决定讨论顺序（★★★★★ 优先）
- **备注列**对于🔄状态必须写明"卡在哪里"，以便下次会话直接续上
- 首次创建时，从 step1 的角色/场景/道具清单批量初始化为 ⬜ 待处理
- 每完成一个元素的讨论，**立即更新状态**，不要等到会话结束再批量更新（防止意外中断丢失进度）

### 一、全局补充设定（可选）

> 对 art_direction.md 的补充和个性化微调。如果导演对全局风格没有额外意见，本节可留空。

可以包含：
- 对 art_direction 某项决策的调整（如"色调再冷一点"、"胶片颗粒感可以更重"）
- 额外的视觉参考（"整体氛围参考《让子弹飞》的乡土感"）
- 附参考图片时，给出路径：`参考图：ref_images/global/xxx.jpg`

**格式**：自由文字 + 可选参考图路径。简洁为主，不需要重复 art_direction 已有的内容。

### 二、角色设定

> 导演对每个角色视觉形象的创意决策。这是本文件最核心的板块。

每个角色用三级标题：`### {角色名}`

每个角色可包含以下字段（全部可选，有的写，没有的留空或不写）：

```markdown
### 宋小仙

**选角偏好**：
精干型，不要太帅，有点沧桑感。眼神要有穿透力但不凶。参考刘烨年轻时的气质。

**参考图**：ref_images/characters/宋小仙_ref.jpg

**阶段划分**：
（如果导演想指定角色的视觉阶段，列在这里。留空则由 step3 从原文提取）
| stage_id | 阶段描述 | 章节范围 |
|----------|---------|---------|
| 民警执勤期 | 警服形象 | ch5-ch9 |
| 便装追踪期 | 便装，风尘仆仆 | ch10-ch27 |

**外貌硬约束**：
（导演明确要求的外貌特征，下游不可更改）
- 单眼皮
- 不戴眼镜（伪装期除外）

**服装偏好**：
警服要旧一点，不要太新太亮。便装偏灰暗色系。

**备注**：
（其他导演想说的）
```

**编写要点**：
- **不需要写完整的外貌描述**——那是 step3 圣经的职责。这里只写导演的"偏好"和"硬约束"
- **可以用参考演员/影视角色来描述气质**——"参考《无间道》里的梁朝伟的疏离感"比写一段外貌描述更有效
- **参考图优先于文字**——如果导演有参考图，一张图胜过千言万语
- **阶段划分是可选的**——导演可以指定（如"我只想要 3 个阶段"），也可以留给 step3 根据原文自动划分
- **留空即委托**——没写的字段意味着"交给 AI 自行决定"，但 AI 需确保与已定义的角色有区分度

### 三、场景设定

> 导演对关键场景视觉形象的创意决策。

每个场景用三级标题：`### {场景名}`

```markdown
### 石岭关

**视觉偏好**：
雾气缭绕的山隘口，两侧是陡峭的石壁。参考张家界的石柱地貌，但更阴森。
不要绿意盎然的感觉，要枯寂、荒凉。

**参考图**：ref_images/locations/石岭关_ref.jpg

**光线氛围**：
永远是阴天或薄雾，不出现阳光直射。

**备注**：
这个场景在故事中是"边界"的象征，视觉上也要有"跨过去就回不来"的感觉。
```

**编写要点**：
- 只写导演有明确想法的场景，不需要覆盖所有场景
- 可以描述氛围而非具体细节——"阴森"、"温暖"、"压抑"都是有效的输入
- 参考图可以是实景照片、其他影视作品截图、甚至 AI 生成的概念图

### 四、道具设定

> 导演对重要道具外观的创意决策。

每个道具用三级标题：`### {道具名}`

```markdown
### 追魂瓶

**外观偏好**：
古朴的青铜小瓶，表面有锈绿色的铜锈。瓶口用红色封蜡密封。
不要太精致，要有年代感和手工感。

**参考图**：ref_images/props/追魂瓶_ref.jpg
```

**编写要点**：
- 只写对叙事有重要意义的道具
- 重点是"这个道具给观众什么感觉"，而非精确的尺寸参数

## 工作流程

> **核心设计**：本流程支持**跨会话中断与恢复**。导演可以今天完成 10 个角色，明天继续 5 个角色和 3 个场景。进度通过 `director_vision.md` 的"〇、进度追踪"表持久化。

### 第一步：初始化或恢复

**首次启动（`director_vision.md` 不存在）**：
1. 读取 step1 产出的角色、场景、道具清单
2. 创建 `director_vision.md`，初始化进度追踪表（所有元素状态为 ⬜ 待处理）
3. 按重要度排序（★★★★★ 在前），角色 → 场景 → 道具的顺序排列
4. 向导演展示清单概览："共 X 个角色、Y 个场景、Z 个道具待设定。建议从主角开始，你想先处理哪些？"

**恢复会话（`director_vision.md` 已存在）**：
1. 读取 `director_vision.md` 的进度追踪表
2. 向导演汇报进度摘要："上次完成了 N 个角色、M 个场景。还剩 A 个角色（其中 B 个🔄进行中）、C 个场景、D 个道具待处理。"
3. 如果有🔄进行中的元素，优先处理："上次 {角色名} 还没确认完，{备注内容}，我们先把它收尾？"
4. 否则从下一个 ⬜ 待处理的元素继续

### 第二步：逐元素与导演交流

按进度表顺序，逐个元素与导演对话。对话中应**主动提供建议和选项**供导演决策，而非让导演从零开始描述。

**每完成一个元素的讨论，立即做两件事**：
1. 将设定内容写入 `director_vision.md` 对应板块（角色/场景/道具）
2. 更新进度追踪表状态为 ✅（或 🔄 如果还有字段未确认）

这样即使会话意外中断，已完成的工作不会丢失。

**角色相关问题**（对每个重要角色逐一询问）：
- 你心目中这个角色是什么类型的人？有没有参考演员或影视角色？
- 有没有参考图片？
- 这个角色在故事中外貌/着装会发生变化吗？需要分几个阶段？
  - 助手应先基于原文分析，**主动提出阶段划分建议**（如"根据原文，宋小仙有 6 次明显的着装变化，建议分为以下阶段：……你觉得需要调整吗？"）
  - 导演可以接受建议、合并阶段（"太多了，合成 3 个就行"）、或自定义划分
- 有没有必须保留的外貌特征（如"必须有疤痕"、"一定是光头"）？
- 有没有必须避免的外貌特征（如"不要太帅"、"不要显老"）？

**场景相关问题**（对导演有想法的场景）：
- 这个场景你有特别的视觉想象吗？参考哪些实景或影视作品？
- 有没有参考图片？
- 光线和氛围有特别要求吗？

**道具相关问题**（对关键道具）：
- 这个道具你想象中是什么样的？
- 有没有参考图片？

**导演想中断时**：
- 导演随时可以说"今天先到这"、"下次继续"等
- 助手应确认当前元素的状态（✅ 还是 🔄），更新进度表，保存文件
- 给出简短的进度总结："今天完成了 X 个角色、Y 个场景。下次还有 A 个角色、B 个场景、C 个道具待处理。"

### 第三步：整理与确认

每个元素在讨论完成后就已实时写入文件，因此本步骤主要是**阶段性回顾**：

- 当一个类别（如角色）全部完成时，将该类别的内容整体呈现给导演做一次确认
- 所有元素全部完成后，做一次完整的最终确认
- 参考图片确认路径是否正确并放入 `ref_images/` 对应子目录

### 第四步：标记完成

所有元素处理完毕且导演确认后：
1. 进度追踪表中所有元素应为 ✅ 或 ⏭️
2. 在"最后更新"行标注"全部完成"
3. 此文件进入只读状态，后续修改需导演明确要求

## 与下游的衔接

### step3（圣经编写）如何使用本文件

1. 开始编写角色圣经前，**先读 `director_vision.md`** 对应角色的设定
2. **阶段划分**：如果导演指定了阶段，直接使用；如果未指定，从原文提取
3. **外貌描写**：以导演的选角偏好和参考图为基础展开描写，而非凭空编造
4. **外貌硬约束**：导演标注的硬约束不可更改
5. **未设定的字段**：从原文提取，原文不足时模型补全，但需确保与已有角色有区分度

### step4（视觉资产生成）如何使用本文件

1. 如果导演提供了参考图，生成肖像时将参考图作为 Seedream 的 reference image 输入
2. 导演的选角偏好可以作为 prompt 的补充约束

## 常见错误（必须避免）

| 错误 | 后果 | 预防 |
|------|------|------|
| 把导演设定写成完整的角色圣经 | 职责重叠，维护两份文件 | 只写偏好和约束，详细描写交给 step3 |
| 导演没有意见时强行编造设定 | 伪造导演意图，限制了模型的合理发挥 | 留空即可，留空意味着"委托" |
| 参考图路径写错 | 下游找不到图片 | 写完后验证路径是否存在 |
| 忘记写阶段划分 | step3 可能划分出与导演预期不同的阶段数量 | 导演有想法的角色务必写上阶段划分 |
| 角色选角偏好太模糊 | "帅一点"对 AI 没有指导意义 | 用参考演员/角色 + 具体特征描述 |
| 讨论完不及时更新进度表 | 会话中断后丢失进度，下次重复讨论 | 每完成一个元素立即更新状态和写入内容 |
| 🔄状态不写备注 | 下次恢复时不知道卡在哪里 | 备注列必须写明未完成的具体字段 |

## 质量标准

合格的导演视觉设定必须满足：

1. **意图清晰**：每条设定都能让下游明确理解导演想要什么
2. **不越界**：只写偏好和约束，不写完整的角色/场景描述
3. **可追溯**：参考图路径有效，文字描述可与导演确认
4. **有区分度**：不同角色的选角偏好之间有明显差异
5. **适度留白**：导演没有想法的留空，不强行填满
