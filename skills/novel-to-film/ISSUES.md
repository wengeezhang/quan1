## 问题 1：角色肖像相似度过高，多个角色"撞脸"

**发现时间**：2026-03-23（step6 keyframe 生成阶段）

**现象**：
使用 Seedream 生成角色肖像和分镜首帧时，不同角色的生成图片外貌高度相似，难以区分。配角尤为严重。

**根因**：
1. **小说原文外貌描写稀缺**：小说是文学作品，不是选角手册。大量角色（尤其配角）在原文中仅有"四十多岁的壮汉"之类的简单描述，缺乏可区分的视觉细节
2. **模型补全趋向通用特征**：step3 编写角色圣经时，面对稀缺的原文信息，模型倾向于用"安全"的默认特征补全（方脸、深邃眼睛、黝黑皮肤），导致多个角色的外貌描写趋同
3. **批量生成缺乏交叉比对**：53 个角色的圣经分批生成，每次生成时模型看不到其他角色已"占用"的外貌特征，导致描述用词高度重叠
4. **身份锚点是事后补救**：`identity_anchors.md` 的 6 维区分矩阵是在圣经写完后做的碰撞检查，发现重复时已难以大规模回改

**影响范围**：step3（角色圣经）→ step4（视觉资产生成）→ step6（分镜首帧）→ step7（视频生成）

**已采取的缓解措施**：
- SKILL.md 常见错误表中新增两条警示（补全前参考已有角色、批量生成后做差异化检查）

**待解决**：
- [ ] 重新审视 53 个角色的身份锚点碰撞情况，量化重复程度
- [ ] 设计更好的生成流程：在编写每个角色外貌时，强制输入已有角色的外貌摘要作为"排除约束"
- [ ] 考虑是否需要对现有角色圣经做一轮外貌差异化修订

## 问题 2：参考图角度与镜头构图冲突，人物被强行转向镜头

**发现时间**：2026-03-23（step6 keyframe 生成阶段）

**现象**：
使用正面半身像作为参考图生成分镜首帧时，即使 prompt 描述人物"在排队面向窗口"，Seedream 仍然将人物转向镜头，破坏构图。多次生成均复现。

**根因**：
1. **参考图的视觉引导力压过文本 prompt**：当参考图是正面照时，模型会强烈倾向于让生成的人物也面向镜头，以"匹配"参考图的角度
2. **原肖像集只有正面/近正面角度**：原 SKILL.md 定义的必选视角是正面半身、3/4半身、侧面半身、全身正面——缺少侧面全身、背面全身等角度，导致生成侧面/背面镜头时无合适参考图可用

**解决方案**：
- [x] 调整肖像视角配置为 5 张：正面半身像（面部锚点）+ 全身正面 + 全身3/4 + 全身正侧面 + 全身背面
- [ ] `generate_keyframes.py` 中增加智能参考图选取逻辑：根据镜头的机位角度和人物朝向，自动匹配最接近角度的参考图，而非每次都用正面照

## 问题 3：Layer 3 英文翻译严重丢失场景细节

**发现时间**：2026-03-23（step6 keyframe 生成阶段）

**现象**：
SC001-001 的中文画面描述详细写了排队栏杆、户政大厅窗口、人物佝偻姿态等具体信息，但翻译后的 Layer 3 变成了抽象的 "ethereal luminescence"、"suspended in moment" 等与原文完全无关的内容。

**根因**：
- 批量翻译（Claude Desktop 对话生成）时，面对大量条目，翻译质量不稳定
- 翻译过程本身是有损的——即使翻译准确，英文 prompt 的表达习惯也会偏离中文原文的具体场景描写
- Seedream 本身原生支持中文，翻译这一步从根本上就是不必要的

**解决方案**：
- [x] 去掉翻译环节，`generate_keyframes.py` 直接使用 `layer3_source_cn`（分镜脚本的中文画面描述）作为 Layer 3
- [x] `layer3_translations.json` 不再需要
- [x] 更新 SKILL.md、README.md、generate_keyframes.py 文档和代码
