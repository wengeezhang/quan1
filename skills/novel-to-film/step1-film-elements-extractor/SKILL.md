# 影视元素提取器

## 定位

本技能的核心目标是**为后续影视可视化制作提供最重要的信息**。

这意味着：提取的不是小说的"文学内容摘要"，而是**能被概念设计师画出来、能被导演拍出来、能被特效团队做出来**的视觉元素。每一条提取结果都应该能回答这个问题："这条信息，视觉制作团队拿到后能用吗？"

本技能处于整个影视制作流程的最上游——它的产出直接决定下游角色圣经、场景设计、分镜脚本等环节的质量和完整性。

## 核心原则

1. **视觉导向**——始终以"这个元素能否产生画面"为提取标准。纯文学性的心理描写、抽象的哲学思辨如果不能转化为视觉表现，不纳入提取范围
2. **先全局，后局部**——必须扫描全部章节后才能输出元素清单，绝不可只读前几章就开始提取
3. **抓大放小**——优先提取视觉冲击力强、出场频率高、对画面有决定性影响的元素；环境背景中一闪而过且无视觉特征的元素可以后置
4. **章节溯源**——每个元素必须标注章节来源，确保下游团队能回溯原文获取更多细节
5. **分类服务于制作分工**——元素分类对应实际的制作团队分工（角色→造型/表演、场景→美术/置景、道具→道具组、事件→分镜/特效），而非文学分析分类

## 输入

```
chapters/
├── part1_chapter1.md              # 原文正文
├── part1_chapter1_character.md    # 章节角色分析（如有）
├── part1_chapter1_summary.md      # 章节摘要（如有）
├── part1_chapter1_summary_simple.md  # 章节简要摘要（如有）
├── ...
└── part5_chapter71.md
```

**优先读取顺序**：
1. `*_summary.md` / `*_summary_simple.md`——快速建立全局认知
2. `*_character.md`——提取角色及其视觉信息
3. `*.md`（正文）——补充视觉细节和验证

## 输出

所有产出文件存放在 `production/step1-extraction/` 目录下：

```
production/step1-extraction/
├── characters.md          # 角色清单（含视觉特征摘要）
├── locations.md           # 场景/地点清单（含空间与氛围描述）
├── props.md               # 关键道具清单（含外观与功能）
├── events.md              # 关键事件清单（按视觉强度排序）
├── world_settings.md      # 世界观设定（有视觉表现的规则体系）
├── timeline.md            # 时间线（关联视觉变化节点）
└── coverage_report.md     # 覆盖率验证报告
```

## 工作流程

### 第一步：快速通读（建立全局认知）

按顺序读取全部 `*_summary_simple.md` 或 `*_summary.md`，建立对整部小说的全局理解：
- 故事总共几个Part，每个Part的视觉基调是什么
- 主线剧情走向及其视觉高潮点
- 大致的角色阵营划分和视觉对立关系

```
# 建议的扫描顺序
chapters/part1_chapter1_summary_simple.md
chapters/part1_chapter2_summary_simple.md
...
chapters/part5_chapter71_summary_simple.md
```

### 第二步：角色提取

逐章扫描 `*_character.md` 和正文，提取所有需要被"画出来"或"演出来"的角色。

**角色清单格式**（`production/step1-extraction/characters.md`）：

```markdown
# 角色清单

## 主角

| 角色名 | 别名/代号 | 身份 | 首次出现 | 末次出现 | 出场章数 | 重要性 | 视觉特征摘要 |
|--------|----------|------|---------|---------|---------|--------|------------|
| 宋小仙 | — | 追魂使 | 第1章 | 第71章 | 71 | ★★★★★ | 女性，表情控制力极强，锁链纹路标识 |

## 配角

（同上格式）

## 群像/一次性角色

（同上格式，可简化）
```

**提取规则**：
- 有视觉出场的角色必须收录（哪怕只有一个画面）
- 同一角色的不同形态/化名必须合并为一条，在"别名/代号"中列出
- **视觉特征摘要**是本清单最重要的字段——用一句话概括该角色最显眼的视觉辨识点，供下游团队快速定位
- 角色分级标准：
  - ★★★★★：贯穿主线，需要完整角色圣经
  - ★★★★☆：关键转折点角色，需要基础视觉设计
  - ★★★☆☆：重要支线角色，需要外貌和着装定义
  - ★★☆☆☆：有面部特写的次要角色
  - ★☆☆☆☆：背景人物，仅需群像风格定义

### 第三步：场景/地点提取

扫描全部章节，提取所有需要被"搭建"或"绘制"的场景。

**场景清单格式**（`production/step1-extraction/locations.md`）：

```markdown
# 场景/地点清单

## 主要场景（多章反复出现）

| 场景名 | 所属区域 | 首次出现 | 末次出现 | 出现章数 | 重要性 | 空间特征 | 氛围关键词 |
|--------|---------|---------|---------|---------|--------|---------|-----------|
| 恶灵区 | 阴间 | 第1章 | 第71章 | 15+ | ★★★★★ | 封闭、黑暗、塔状结构 | 压抑、沉重、冰冷 |

## 次要场景

（同上格式）

## 一次性场景

（同上格式，可简化）
```

**提取规则**：
- 有具体空间描写的场景必须收录
- **空间特征**（室内/室外、开阔/封闭、明/暗、关键建筑结构）和**氛围关键词**是核心字段，直接服务于美术和灯光部门
- 同一地点在不同章节如果视觉面貌有重大变化（如白天vs夜晚、战前vs战后），须标注

### 第四步：道具提取

提取所有需要被"制作"或"渲染"的物品。

**道具清单格式**（`production/step1-extraction/props.md`）：

```markdown
# 关键道具清单

| 道具名 | 类型 | 所属角色 | 首次出现 | 功能/意义 | 外观描述 | 重要性 |
|--------|------|---------|---------|----------|---------|--------|
| 泉水 | 超自然物质 | — | 第X章 | 增强灵魂力量 | 清澈蓝白色液体，发光 | ★★★★★ |
| 小顶的围巾 | 个人物品 | 小顶→朱围庸 | 第X章 | 情感纽带/角色标识 | 需定义颜色和材质 | ★★★★☆ |
```

**提取规则**：
- 筛选标准：这个道具是否需要在画面中被看到？如果是，收录
- 有超自然视觉效果的物品/能量优先级最高（需要特效制作）
- 反复出现或承载情感象征的物品优先级次之（需要道具组制作）
- **外观描述**字段：尽可能记录原文中的视觉描写，无描写的标注"需定义"

### 第五步：关键事件提取

提取需要被"拍出来"的重大视觉场景。

**事件清单格式**（`production/step1-extraction/events.md`）：

```markdown
# 关键事件清单

| 事件名 | 章节 | 涉及角色 | 事件类型 | 视觉强度 | 制作难度 | 视觉要点 |
|--------|------|---------|---------|---------|---------|---------|
| 朱围庸首次占身 | 第4章 | 朱围庸、丁路 | 转折点 | ★★★★★ | 高（特效） | 紫色光团侵入人体，原主灵魂被驱逐 |
```

**事件类型**：转折点、高潮、揭示、决战、牺牲、重逢、背叛、觉醒

**核心字段说明**：
- **视觉强度**：该事件被制作成画面时的冲击力等级（1-5星），是分镜师和导演排优先级的依据
- **制作难度**：低（实拍即可）/ 中（需简单特效）/ 高（需复杂特效或大场面）
- **视觉要点**：一句话描述这个事件"看起来是什么样的"，为分镜设计提供方向

### 第六步：世界观设定提取

提取有视觉表现的规则体系——只提取那些需要在画面中被呈现出来的世界观规则。

**世界观格式**（`production/step1-extraction/world_settings.md`）：

```markdown
# 世界观设定

## 视觉规则体系
（需要在画面中被表现的规则，如：灵魂是什么样子的、占身过程的视觉表现、泉水的效果等）

## 阵营与视觉对立
（不同阵营的视觉区分方式，如：阳间系角色的色调 vs 恶灵系角色的色调）

## 超自然能力视觉体系
（各类超自然能力的视觉呈现规范，如：紫色=恶灵能量、蓝白色=泉水能量）

## 空间体系
（阳间/阴间/恶灵区等不同空间的视觉差异和转换规则）
```

**提取规则**：
- 只收录对视觉制作有直接影响的设定
- 纯剧情逻辑但不产生画面差异的设定不收录（如"谁的官职比谁大"）
- 重点关注颜色体系、光效体系、空间转换规则等视觉语言

### 第七步：时间线梳理

将全部事件按时间顺序排列，标注视觉变化节点。

**时间线格式**（`production/step1-extraction/timeline.md`）：

```markdown
# 故事时间线

| 时间节点 | 章节 | 事件 | 涉及角色 | 视觉变化 |
|---------|------|------|---------|---------|
| 数百年前 | 背景 | 朱围庸被判入恶灵区 | 朱围庸 | 人→灵魂光团 |
| 故事开始 | 第1章 | ... | ... | ... |
```

**视觉变化**字段：标注该时间节点上角色外貌、场景面貌、道具状态等发生的视觉变化，为造型和美术部门提供"什么时候该换装/换景"的时间锚点。

### 第八步：覆盖率验证

生成验证报告，确保无遗漏。

**验证报告格式**（`production/step1-extraction/coverage_report.md`）：

```markdown
# 覆盖率验证报告

## 章节覆盖
- 总章节数：71
- 已扫描章节数：71
- 覆盖率：100%

## 元素统计
- 角色总数：XX（主角X + 配角X + 群像X）
- 场景总数：XX（主要X + 次要X + 一次性X）
- 道具总数：XX
- 关键事件总数：XX（其中视觉强度★★★★★的XX个）

## 视觉制作就绪度检查
- [ ] 每个★★★★★角色都有视觉特征摘要
- [ ] 每个★★★★★场景都有空间特征和氛围关键词
- [ ] 每个★★★★★事件都有视觉要点描述
- [ ] 世界观设定中的颜色/光效体系已提取
- [ ] 时间线中的视觉变化节点已标注

## 待确认项
（列出提取过程中的疑问点，如：某场景缺乏空间描写需要原创设计、某道具外观原文未描述等）
```

## 常见错误（必须避免）

| 错误 | 后果 | 预防 |
|------|------|------|
| 只读前几章就开始提取 | 遗漏后半部分的新角色和新场景 | 必须先通读全部 summary |
| 提取文学信息而非视觉信息 | 下游团队拿到后无法转化为画面 | 每条信息都问"能画出来吗？" |
| 同一角色多次录入 | 下游创建多份重复角色圣经 | 合并别名，建立角色-别名映射 |
| 不标注章节范围 | 下游编写角色圣经时覆盖不完整 | 每个元素必须标注首次/末次 |
| 场景只记名字不记空间特征 | 美术部门无法开始设计 | 必须填写空间特征和氛围 |
| 事件只记剧情不记视觉 | 分镜师不知道该画什么 | 必须填写视觉要点 |

## 质量标准

合格的元素提取必须满足：

1. **视觉可用**：每条提取结果都包含视觉制作团队可直接使用的信息
2. **全覆盖**：扫描了100%的章节，无遗漏
3. **可溯源**：每个元素都标注了章节来源
4. **无重复**：同一元素只出现一次（别名已合并）
5. **分类清晰**：分类对应制作团队分工，角色/场景/道具/事件互不交叉
6. **分级合理**：重要性评级与视觉出场频率和画面权重匹配
