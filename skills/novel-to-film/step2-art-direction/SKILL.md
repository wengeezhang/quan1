# Art Direction（全局视觉风格定义）

## 定位

本技能的核心目标是**为整部影片建立统一的视觉语言**。

在好莱坞制作流程中，这对应 Look Development / Art Direction 阶段——它发生在概念设计之前，确保后续所有视觉产出（角色设计、场景设计、道具设计、分镜、首帧、视频）共享同一套视觉基因。

本技能处于整个流程的第二阶段。它的上游是元素提取（阶段1），下游是所有圣经编写（阶段3）和视觉资产生成（阶段4）。**没有这一步，各元素的视觉风格会各自为政，导致最终成片画风不统一。**

在 AI 制作流中，Art Direction 的产出还有一个特殊作用：它将被提炼为**全局 style prompt 前缀**，作为后续所有 Seedream/Seedance 生成时的强制前缀，从技术层面保证画风一致。

## 核心原则

1. **统领全局**——Art Direction 不为任何单一角色或场景服务，它定义的是整部影片的视觉基因，所有下游产出都必须遵守
2. **可执行，不空谈**——每一项决策都必须落实到 AI 生成时可直接使用的具体描述，而非抽象的美学概念（如"色调偏暖"不够，需要写成"主色调 #2C1810 深褐、辅助色 #8B4513 鞍褐、高光 #FFD700 金色"）
3. **从原著中来**——视觉风格必须从小说的文字氛围、时代设定、世界观中提炼，不能脱离原著凭空创造
4. **留有弹性**——Art Direction 定义的是"基线"和"边界"，不是僵硬的规则。不同场次、不同情绪段落允许在基线上做合理偏移，但偏移范围也需要预先定义
5. **可验证**——产出的每一项定义都能用来检验后续产出是否合规（如"这张角色图的色调是否在定义的色彩范围内？"）

## 输入

- `production/step1-extraction/characters.md`——角色清单（了解角色阵营和视觉对立关系）
- `production/step1-extraction/locations.md`——场景清单（了解主要空间类型和氛围需求）
- `production/step1-extraction/world_settings.md`——世界观设定（了解超自然视觉体系、空间体系）
- `production/step1-extraction/events.md`——关键事件清单（了解视觉高潮点的氛围需求）
- `chapters/`——原始小说章节（回溯原文提炼整体氛围）

**研究方法**：
1. 先读元素清单（characters、locations、world_settings），建立对故事视觉世界的整体认知
2. 读 events.md，识别不同情绪段落的分布
3. 选读关键章节原文，感受原著的文字氛围和时代质感
4. 综合提炼视觉风格定义

## 输出

一份 Markdown 文件，存放在 `production/step2-art-direction/` 下：

```
production/step2-art-direction/
└── art_direction.md          # 全局视觉风格定义
```

## 文件结构（七章制，顺序固定）

```markdown
# [影片名/项目名] | Art Direction

## 一、整体风格定位
## 二、色彩体系
## 三、光影风格
## 四、时代与质感
## 五、构图与镜头语言
## 六、阵营视觉体系
## 七、全局 Style Prompt 前缀
```

每个章节之间用 `---` 分隔线隔开。

## 各章节编写规范

### 一、整体风格定位

> 确定整部影片的视觉大方向，须包含：
> 1. **风格光谱定位**——在"写实 ← → 风格化"光谱上，本片处于什么位置？给出精确描述（如"80%写实 + 20%超自然风格化"）
> 2. **视觉参考影片**——列出 3-5 部视觉风格最接近的影视作品，说明参考的具体方面（色调、光影、构图、特效风格等）
> 3. **总体氛围关键词**——5-8 个定义全片视觉氛围的关键词
> 4. **视觉禁区**——明确列出本片绝对不应出现的视觉风格（如"不使用日系动漫风格的夸张表情"、"不使用赛博朋克霓虹光效"）

**格式**：
- 风格光谱用文字图示：`写实 ●────────○ 风格化` 标注本片位置
- 视觉参考影片用列表：`- 《影片名》——参考其XXX方面`
- 总体氛围关键词用逗号分隔的一行
- 视觉禁区用列表

### 二、色彩体系

> 定义全片的色彩语言，须包含：
> 1. **主色调**——全片最主要的 2-3 种颜色（含 HEX 色值），说明使用场景和情感含义
> 2. **辅助色**——补充色 2-3 种（含 HEX 色值），说明在什么情境下使用
> 3. **强调色/警示色**——用于关键时刻的高饱和度颜色（含 HEX 色值）
> 4. **禁用色**——全片中不应出现的颜色（含原因）
> 5. **情绪-色彩映射**——不同情绪段落/情节类型对应的色彩倾向（表格形式）
> 6. **色彩过渡规则**——从一种色调切换到另一种时，是否允许硬切，还是必须通过渐变过渡

**格式**：
- 主色调/辅助色/强调色/禁用色用列表，每项包含色块标注（`■ #XXXXXX 颜色名`）和说明
- 情绪-色彩映射用表格：`| 情绪/情境 | 色彩倾向 | 饱和度 | 示例场景 |`

### 三、光影风格

> 定义全片的光影语言，须包含：
> 1. **默认光影模式**——全片大部分场景使用的光影风格（如"自然光为主，侧光为默认光源角度"）
> 2. **场景类型-光影映射**——不同类型的场景对应的光影处理方式（表格形式）
> 3. **超自然/特效光影**——超自然能力、特殊事件的光效规范（颜色、强度、扩散方式、对环境的影响）
> 4. **阴影处理**——阴影的硬度、密度、颜色倾向
> 5. **对比度基线**——全片的整体对比度倾向（低对比柔和 / 中等对比 / 高对比戏剧性）

**格式**：
- 默认光影模式用文字描述
- 场景类型-光影映射用表格：`| 场景类型 | 主光源 | 光质 | 阴影特征 | 整体亮度 | 参考 |`
- 超自然光影用列表，每项描述颜色、强度和表现方式

### 四、时代与质感

> 定义影片的时代感和材质质感，须包含：
> 1. **时代定位**——故事发生的时代/年代，以及在视觉上如何体现这个时代
> 2. **材质偏好**——画面中常见材质的质感方向（如"偏向粗糙天然材质——石材、粗布、生铁"而非"光滑现代材质——玻璃、不锈钢、塑料"）
> 3. **建筑风格**——主要建筑的视觉风格特征
> 4. **服饰风格**——服装的总体风格方向（面料质感、剪裁风格、装饰程度）
> 5. **画面质感/胶片感**——画面本身的质感方向（如"胶片颗粒感"、"数字清晰风"、"油画质感"等）

**格式**：
- 各子项用加粗列表：`- **时代定位**：...`
- 有对比时用"✓ 应有 / ✗ 应避免"格式

### 五、构图与镜头语言

> 定义全片的构图偏好和镜头语言，须包含：
> 1. **画面比例**——宽银幕（2.39:1）/ 标准宽屏（16:9）/ 其他，以及选择原因
> 2. **构图偏好**——对称/非对称、空间留白方向、角色在画面中的位置倾向
> 3. **景深偏好**——默认的景深处理（浅景深/深景深/随场景变化）
> 4. **运镜风格**——默认的运镜节奏（稳重缓慢 / 手持跟拍 / 固定机位为主），以及特殊情境的运镜变化
> 5. **特殊镜头语言**（可选）——如果原著有特殊的叙事手法需要对应的镜头语言（如闪回、多视角叙事）

**格式**：
- 各子项用加粗列表
- 运镜风格可用表格：`| 情境 | 默认运镜 | 景别倾向 | 节奏 |`

### 六、阵营视觉体系

> 如果故事涉及多个阵营/势力/空间，定义各阵营的视觉区分方式。须包含：
> 1. **阵营列表**——各阵营名称及其视觉核心特征（一句话）
> 2. **阵营色彩对照**——每个阵营的代表色及其色值
> 3. **阵营空间特征**——每个阵营所处空间的视觉特征（建筑、光线、材质）
> 4. **跨阵营的视觉对立**——当不同阵营的角色同处一个画面时，如何通过视觉手段体现对立（色温对比、光影方向、景深分离等）

**格式**：
- 阵营对照用表格：`| 阵营 | 代表色 | 光影 | 材质 | 氛围关键词 |`
- 视觉对立用具体的构图/调色描述

**适配规则**：
- 如果故事没有明确的阵营对立，本章改为"空间视觉体系"，按不同空间/世界定义视觉差异
- 如果都没有，可省略本章

### 七、全局 Style Prompt 前缀

> 将前六章的核心视觉决策，提炼为一段可直接复用的 AI 生成 prompt 前缀。须包含：
> 1. **通用前缀**——适用于所有生成（角色、场景、道具）的基础风格描述
> 2. **角色生成前缀**——在通用前缀基础上追加的角色相关风格约束
> 3. **场景生成前缀**——在通用前缀基础上追加的场景相关风格约束
> 4. **特效/超自然生成前缀**——在通用前缀基础上追加的特效相关风格约束

**格式**：
- 每个前缀必须**中英双语**输出——英文版（用于 Seedream/Seedance 生成）和中文版（英文版的直译，用于人工审核），分别用 `**English：**` 和 `**中文：**` 标注，各用独立代码块包裹
- 标注使用场景（"用于 Seedream 生成时作为 prompt 开头"）

**编写要点**：
- 前缀必须是 AI 可直接解析的自然语言描述，不能是抽象概念
- 前缀长度控制在 50-100 词以内（过长会稀释具体提示词的效果）
- 通用前缀应包含：风格定位、画面质感、主色调倾向、光影风格
- 前缀中不应包含具体角色或场景信息——那是各元素圣经中 AI 提示词的职责

## 工作流程

### 第一步：全局认知建立

读取阶段1产出的全部元素清单（`production/step1-extraction/` 下的 characters.md、locations.md、world_settings.md、events.md），建立对故事视觉世界的整体认知：
- 故事的世界观类型（现实 / 奇幻 / 科幻 / 混合）
- 主要空间类型和它们之间的视觉对比
- 阵营/势力的视觉对立关系
- 视觉高潮点的数量和类型

### 第二步：原著氛围提炼

选读 3-5 个关键章节的原文，感受原著的文字氛围：
- 作者偏好什么样的环境描写（阴暗压抑？明朗开阔？）
- 超自然元素的视觉描写倾向（华丽特效？克制暗示？）
- 情绪变化的节奏和幅度
- 时代背景的具体线索（建筑、服饰、器物）

### 第三步：视觉参考收集

基于前两步的认知，确定 3-5 部视觉风格最接近的影视参考作品，明确每部参考的具体方面。这一步为后续决策提供直观的视觉锚点。

### 第四步：按章节编写

严格按七章顺序编写，每章完成后确认：
- 是否足够具体、可执行？（能否直接用于检验下游产出）
- 是否与前一章的决策一致？（如色彩体系与光影风格是否自洽）
- 是否有遗漏的重要决策？

### 第五步：Style Prompt 提炼与验证

将前六章的核心决策压缩为 style prompt 前缀，然后用以下方式验证：
- 将通用前缀 + 一个角色描述，想象生成结果是否符合整体风格
- 将通用前缀 + 一个场景描述，想象生成结果是否符合整体风格
- 前缀是否简洁到 AI 能有效解析（不超过 100 词）

## 常见错误（必须避免）

| 错误 | 后果 | 预防 |
|------|------|------|
| 只写抽象概念不给具体参数 | 下游无法执行，各自理解不同 | 每项决策都要有色值/参考影片/具体描述 |
| 风格定义脱离原著 | 视觉与故事脱节，观众违和 | 每项决策都回溯原文依据 |
| style prompt 过长 | 稀释具体提示词效果，AI无法有效解析 | 控制在 50-100 词 |
| 没有定义"禁区" | 下游可能偏离风格而不自知 | 每个章节都列出"不应有"的内容 |
| 色彩体系不分情境 | 全片色调单一无变化 | 必须定义情绪-色彩映射 |
| 忽略超自然光效规范 | 特效风格各镜头不一致 | 单独定义超自然/特效光影 |
| 没有定义阵营视觉对立 | 不同势力在画面上分不清 | 阵营体系单独成章 |

## 质量标准

合格的 Art Direction 必须满足：

1. **可执行**：每一项决策都包含足够具体的参数（色值、参考影片、具体描述），下游团队无需再猜测
2. **自洽**：色彩体系、光影风格、时代质感、构图偏好之间互不矛盾
3. **可验证**：任何一张后续生成的图片，都能用本文件检验其是否"合规"
4. **有弹性**：定义了基线和边界，允许在范围内的合理变化，而非一刀切
5. **源于原著**：每项核心决策都能追溯到原著的文字氛围或世界观设定
6. **prompt 可用**：全局 style prompt 前缀能直接粘贴到 Seedream 生成流程中使用
