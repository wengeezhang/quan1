# 小说转电影制作资料

## 核心原则

**先全局，后局部** — 必须先完整扫描全部章节，建立完整元素清单，再逐一创建资料文件。

## 工作流程

### 阶段1：全局扫描（必须）

在创建任何文件之前，先扫描全部章节提取元素清单：

```bash
# 提取所有地名
grep -rh "城\|镇\|村\|山\|森林\|河\|湖\|桥" chapters/ | sort | uniq

# 提取所有人名（需根据具体小说调整）
grep -rh "老\|小\|先生\|夫人" chapters/ | sort | uniq

# 提取关键道具/物品
grep -rh "泉水\|灵魂\|宝物" chapters/ | sort | uniq
```

产出：`_extraction/locations.txt`, `_extraction/characters.txt`, `_extraction/props.txt`

### 阶段2：元素分类与章节标注

为每个元素标注出现章节范围：

| 元素 | 类型 | 首次出现 | 末次出现 | 重要性 |
|------|------|----------|----------|--------|
| 荣康城 | 场景 | 第46章 | 第70章 | ★★★★★ |
| 小顶 | 角色 | 第10章 | 第71章 | ★★★★☆ |

**重要性评估标准**：
- ★★★★★：贯穿主线，多章出现
- ★★★★☆：关键转折点出现
- ★★★☆☆：支线重要元素
- ★★☆☆☆：偶尔出现
- ★☆☆☆☆：一次性出现

### 阶段3：建立文件夹结构

```
production/
├── 01_story_bible/      # 世界观、时间线
├── 02_characters/       # 角色圣经
│   ├── 主角/
│   ├── 配角/
│   └── 群像/
├── 03_locations/        # 场景圣经
│   ├── 阳间/
│   └── 阴间/（如适用）
├── 04_props/            # 关键道具
├── 05_visual_design/    # 色彩脚本、光影设计
├── 06_key_scenes/       # 高光时刻、AI提示词
└── 07_continuity/       # 连续性追踪
```

### 阶段4：按优先级创建文件

创建顺序：★★★★★ → ★★★★☆ → ★★★☆☆ → ...

每个文件必须覆盖元素的**完整章节范围**，不可只覆盖部分。

### 阶段5：交叉验证（必须）

创建完成后执行验证：

```bash
# 验证场景覆盖
for loc in $(cat _extraction/locations.txt); do
  if [ ! -f "production/03_locations/*/$loc.md" ]; then
    echo "缺失: $loc"
  fi
done
```

验证检查清单：
- [ ] 所有★★★★★元素已创建文件
- [ ] 每个文件覆盖元素的完整章节范围
- [ ] 无遗漏的重要场景/角色/道具

## 常见错误（必须避免）

| 错误 | 后果 | 预防 |
|------|------|------|
| 只读Part 1就开始创建 | 遗漏后续章节的元素 | 先扫描全部章节 |
| 边做边补 | 覆盖不完整 | 先建清单再创建 |
| 不验证覆盖率 | 遗漏难以发现 | 必须执行阶段5 |
| 假设早期章节包含全部 | 错过后期新增元素 | 全局扫描无例外 |

## 角色圣经编写规范

角色圣经是为视觉制作（概念设计、AI出图、特效、导演、演员）服务的角色完整档案。每份角色圣经由以下八个章节构成，章节顺序固定，编号从一到八。

### 整体原则

- **服务对象**：概念设计师、AI绘图、特效团队、导演、演员、造型师、调色师
- **核心定位**：定义"这个角色是什么样的人"，而非"这个镜头怎么拍"（分镜指导属于导演手册，不放在角色圣经中）
- **编写规范的作用**：每个章节开头的编写规范是模板指令，告知编写者该章节必须包含哪些内容、以什么格式呈现。编写规范用 blockquote（`>`）格式书写，放在章节标题之后、正文内容之前
- **章节内不要混入其他章节的内容**：例如外貌章节不写性格分析，能力章节不写人物关系

### 文件结构

```
# [角色名] | 角色圣经

## 一、基本信息
## 二、外貌与形态
## 三、能力与视觉表现
## 四、角色发展轨迹
## 五、人物关系
## 六、视觉设计
## 七、连贯性约束
## 八、AI绘图提示词
```

### 一、基本信息

> **编写规范**：本章节以表格形式提供角色的核心档案信息，须包含以下字段：
> 1. **角色名**——角色的正式名称（如有别名、代号、绰号，一并列出）
> 2. **初期身份**——角色首次出场时的社会身份或职业（观众/读者最初认知的身份）
> 3. **真实身份**——角色的隐藏身份或本质身份（若无隐藏身份则与初期身份一致）
> 4. **后期身份**——角色在故事发展中最终承担的角色定位
> 5. **年龄**——角色的年龄或年龄范围
> 6. **首次出现**——角色首次登场的章节编号
> 7. **覆盖章节**——角色出现的章节范围及总出场章数
> 8. **重要性**——用★标注（1-5星），反映角色在整体叙事中的权重

**格式要求**：使用两列表格（属性 | 描述），每个字段加粗属性名。

### 二、外貌与形态

> **编写规范**：本章节为角色的视觉呈现提供完整的制作参考，须包含以下内容：
> 1. **基础外貌**——按部位（身材体型、脸型、眉毛、眼睛、鼻子、嘴唇、皮肤、发型等）逐一描述，提供足够细节供概念设计师和AI出图使用。附带标志性微动作和表情管理特征
> 2. **各阶段外貌与着装**——按故事阶段分别描写，每个阶段包含"外貌特征"（面部状态变化：胡茬、肤色、消瘦程度、眼神等）和"衣着与气质"（具体服装描述、关键道具、气质关键词），确保角色在不同阶段的视觉差异清晰可辨，同时阶段之间有合理的过渡

**格式要求**：
- 基础外貌使用分部位列表（如 `- **身材体型**：...`），每个部位提供可出图的具体描述
- 各阶段外貌使用四级标题（`#### 阶段名（章节范围）`），每阶段内分"外貌特征"和"衣着与气质"两块

### 三、能力与视觉表现

> **编写规范**：本章节为每项能力提供完整的制作参考。每项能力须包含以下两部分：
> 1. **机制解析**——说明能力的本质、前置条件、触发方式、强度层级与限制，确保编剧和导演理解"这个能力是什么、怎么用、有什么代价"
> 2. **过程视觉化**——将能力的施展过程拆解为可拍摄/可制作的阶段（如"启动→过程→完成→后续反应"），每个阶段描述具体的肢体动作、光效颜色、环境变化、对施展者和目标的影响，确保概念设计师、特效团队和分镜师能直接据此工作
>
> 非超自然能力（如执法、战术）同样需要视觉化——重点描写施展时的行为模式、肢体语言和标志性动作，为表演和镜头设计提供参考。

**格式要求**：
- 每项能力使用三级标题（`### X.N 能力名称`）
- 能力内部分"机制解析"和"过程视觉化"两个子块
- 过程视觉化建议使用编号步骤（1→2→3→...），每步描述画面

**注意**：如果角色没有超自然能力，本章节可改为"行为模式与标志性动作"，聚焦角色的行为特征和肢体语言。

### 四、角色发展轨迹

> **编写规范**：本章节完整呈现角色从首次出场到终局的演变过程，须包含以下内容：
> 1. **整体演变曲线**——用简明的图示或一句话概括角色从起点到终点的核心弧线，让读者一眼把握角色的整体走向
> 2. **分阶段详解**——按故事阶段逐一展开，每个阶段须包含四个维度：核心特质（这个阶段他"是谁"）、行为表现（具体可演绎的言行举止）、内心驱动（驱动行为的深层动机）、视觉状态（这个阶段他"看起来什么样"，包括外貌、着装、光影氛围的变化）
> 3. **关键转折点**——列出改变角色走向的具体事件，标注转折前后的变化和标志性时刻，为分镜师标记高光场景提供依据
> 4. **视觉叙事线索**（可选）——如果角色存在某条贯穿全剧的视觉特征变化（如表情控制力、眼神颜色、特定道具的出现与消失等），单独提炼出来，说明其叙事功能和制作要点

**格式要求**：
- 整体演变曲线使用代码块（````）画简易文字图示，下方用一句话总结核心弧线
- 分阶段详解使用四级标题（`#### 阶段名（章节范围）`），每阶段内使用加粗列表（`- **核心特质**：...`）呈现四个维度
- 关键转折点使用编号列表，每个转折标注章节和标志性时刻
- 视觉叙事线索使用代码块画变化曲线，附带制作提示

**与已删除章节的关系**：本章节合并了原"身份演变"和"性格演变"的内容，通过四维度格式（核心特质+行为表现+内心驱动+视觉状态）同时覆盖身份、性格和视觉三条线，避免信息分散在多个章节中。

### 五、人物关系

> **编写规范**：本章节描述角色与其他重要角色之间的关系，每组关系须包含以下内容：
> 1. **关系定位**——用一句话概括两人的关系本质（如"搭档"、"追捕者与被追者"、"师徒"等）
> 2. **关系演变**——若两人关系在故事中发生显著变化，须按阶段列出演变过程（可使用表格），标注对应章节和每个阶段的关系特征
> 3. **互动特征**——描述两人相处时的具体行为模式和互动方式，提供可演绎的细节（如对话风格、肢体语言、空间距离等）
> 4. **情感层面**——说明角色对对方的内在情感态度，以及这种情感如何影响其行为决策
>
> 关系按叙事重要性排序，核心关系（有阶段演变的）使用表格呈现，次要关系使用列表即可。每组关系的描写应服务于角色塑造——不是罗列事件，而是揭示"这段关系如何改变了他"。

**格式要求**：
- 每组关系使用三级标题（`### 与XXX`）
- 核心关系（关系有多阶段演变的）：先用表格列出阶段演变（阶段 | 关系 | 特征），再用文字补充互动特征和情感层面
- 次要关系：使用加粗列表（`- **关系定位**：...`、`- **互动特征**：...`、`- **情感层面**：...`）

### 六、视觉设计

> **编写规范**：本章节为美术、造型和后期部门提供角色的整体视觉方案，须包含以下内容：
> 1. **着装演变**——按故事阶段列出角色的服装变化，每个阶段须注明着装描述、关键道具和视觉关键词，使用表格呈现以便造型部门快速查阅
> 2. **颜色基调**——按阶段定义角色的主色调及其叙事含义，为调色、灯光和美术部门提供统一的色彩语言
> 3. **眼神演变**——用一条时间线串联角色在各阶段的眼神特征变化，为演员表演和特写镜头提供方向
> 4. **气质参考**——为每个阶段提供具体的影视角色参考（注明作品名+角色名+气质关键词），让导演和演员快速建立角色感觉；参考应跨越不同影片风格，避免单一来源

**格式要求**：
- 着装演变使用四列表格（阶段 | 着装 | 关键道具 | 视觉关键词）
- 颜色基调使用三列表格（阶段 | 主色调 | 含义），如有精确色值（RGB/HEX）应标注
- 眼神演变使用箭头连接的一行式时间线（`**关键词**（章节）→ **关键词**（章节）→ ...`）
- 气质参考使用加粗列表，格式为 `- **阶段名**：《影片名》中XXX的XXX——一句话描述气质`

### 七、连贯性约束

> **编写规范**：本章节列出跨场景、跨阶段必须保持一致的视觉与行为规则，确保角色在全剧中的视觉统一性和叙事连贯性。每条约束须说明"什么不能变"和"什么可以变"，为导演、美术、特效和剪辑提供明确的底线。

**格式要求**：
- 按约束类别分子章节，常见类别包括：视觉标识的一致性、行为模式的一致性、表情/情绪弧线约束、能力使用的代价一致性
- 每个子章节内使用"不变的/可变的"对照格式：
  ```
  - **不变的**：[必须始终保持一致的要素]
  - **可变的**：[允许随阶段变化的要素及其变化规则]
  ```
- 如有必要，增加"禁忌"条目说明绝对不可出现的情况

**注意**：连贯性约束是角色圣经的"底线文件"，其他章节描述"角色是什么样"，本章节规定"角色绝对不能变成什么样"。

### 八、AI绘图提示词

> **编写规范**：本章节为每个关键形态/阶段提供可直接使用的AI绘图提示词（英文），供概念设计师和AI出图工作流直接调用。

**格式要求**：
- 每个提示词使用三级标题标注形态名称（`### 提示词N：形态名`）
- 提示词使用代码块包裹，便于复制
- 提示词须为英文，包含：人物特征、服装、动作/姿势、环境、光影氛围、画面风格
- 提示词数量应与"角色发展轨迹"中的阶段数对应，确保每个关键阶段都有可出图的提示词
- 不要在提示词中包含角色真名，使用通用描述（如 "Chinese man" 而非角色名）

## 质量标准

合格的制作资料必须满足：

1. **完整性**：覆盖元素出现的全部章节
2. **可用性**：包含可直接使用的AI绘图提示词
3. **一致性**：同一元素在不同文件中描述一致
4. **可追溯**：标注信息来源章节
5. **无冗余**：每条信息只出现在最合适的章节中，不跨章节重复
6. **可制作**：所有描述最终服务于视觉制作，避免纯文学性描写
