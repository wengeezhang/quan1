## 待解决的技术问题

- [ ] **多角色同框场景的生图方案** 🔴 高优
  - 问题：影视中大量场景涉及多个重要角色同框（群戏、会议、对峙等），当前 pipeline 的角色参考图体系是"单人参考"模式，无法处理：
    - 身份不串：多角色同时出现时，生图模型容易将 A 的服装/脸型混到 B 身上
    - 空间站位：谁在左、谁在中、谁背对镜头——纯文字 prompt 难以精确控制
    - 交互关系：人物之间的对话朝向、视线交汇、肢体互动，文字描述精度不够
    - 参考图分配：如何让模型知道"这张参考图对应画面中左边那个人，那张对应右边那个人"
  - 可能的技术方向（待调研）：
    - 分层合成：先单独生成每个人，再合成到同一场景
    - 布局控制：用 bounding box / pose skeleton 指定每个人的位置和姿态
    - Inpainting 逐人绘制：先生成场景，再逐区域填入不同角色
    - ControlNet + 多参考图：通过 pose/depth 条件控制 + 分区参考图
    - 商业方案调研：Seedream / Kling / 可灵等是否已支持多角色一致性
  - **即梦/Seedance 2.0 的 @引用方案**（已初步调研）：
    - Seedance 2.0（2026.2 发布）支持"全方位参考"模式，最多上传 **12 个参考文件**（9图+3视频+3音频）
    - 上传后系统自动分配标签 `@Image1`、`@Image2`…，在 prompt 中用 `@` 引用
    - **多角色 prompt 写法**：`@Image1 作为角色A的面部特征参考，@Image2 作为角色B的面部特征参考，角色A（描述）站在画面左侧，角色B（描述）站在画面右侧`
    - 参考维度可设为"角色特征"，参考强度建议 70%（>80%姿势僵硬，<50%五官变形）
    - 导入多个人物参考图时，参考维度自动切换为"人物合影"模式
    - **核心价值**：这是目前唯一支持在 prompt 中显式区分多个角色参考图的商业方案（Sora/Kling/Veo 均无同等能力）
    - **待验证**：实际多人同框效果（身份串扰程度）、空间站位控制精度、与我们 pipeline 的集成方式
  - 影响范围：step5（分镜脚本）、step6（首帧生成）、step7（视频生成）
