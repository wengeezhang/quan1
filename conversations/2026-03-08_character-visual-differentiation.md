# 2026-03-08 角色视觉区分度讨论

## 背景

在 review step4 视觉资产（榕泉项目，118个资产已全部生成）时，发现多个角色的面部特征高度相似，后续镜头中难以区分。

## 问题分析

以宋小仙和朱国庸为例，两人的外貌描述 prompt 在多个关键维度上趋同：
- 都是「棱角分明」的脸型
- 都是「薄唇抿成一条线」
- 都是「眼神穿透力强」
- 都是「鼻梁挺直」
- 甚至标志性微动作都重复了（右手食指轻叩桌面）

此外，朱国庸描述为30-40岁壮实男子，但生成结果却是60+老者，说明 prompt 的核心特征没有被模型正确捕捉。

## 初步方案：角色视觉分配矩阵

提出在 5 个核心维度（脸型/体型/肤色/年龄感/发型）上预先为所有角色分配差异化特征，确保任意两角色至少 3 维有明显差异，并用标志物（疤痕、胡须、配饰等）兜底。

## 核心结论：人物外貌必须导演介入

经讨论达成共识：

1. **AI 生成的角色外貌 prompt 天然有局限** — 文字到视觉是一对多映射，AI 不知道导演脑中的具体形象
2. **角色间的化学反应是叙事决策** — 主角和反派该不该长得像、搭档该一高一矮，这不是参数优化问题
3. **每个导演对人物有自己的偏好** — 这是主观审美判断，AI 无法替代

**最终方案：**
- AI 的角色是「选角助理」：提取关键词 → 生成初版 prompt → 批量出候选图 → 撞脸预警
- 导演的角色是「拍板人」：从候选中选方向 → 调整/覆写关键特征 → 最终确认
- 流水线改动：在 step3 → step4 之间增加**导演审核节点**

## 深入探讨：如何系统性保障全剧人物区分度

### 方案一：参考演员锚定法（最推荐）

借鉴真实电影选角思路——导演不描述五官，而是说"我要一个年轻梁朝伟的感觉"。

- 每个角色指定 1-2 个参考演员作为视觉锚点（不是生成其本人）
- 例：宋小仙 → "张震的冷峻 + 彭于晏的精干"；朱国庸 → "姜武的粗犷 + 廖凡的沉默感"
- 优势：参考演员本身高度差异化；模型对明星脸的理解远比抽象描述精确；导演介入成本极低（选演员比逐条改五官快10倍）
- 落地：step3 角色圣经增加「视觉参考」字段，导演填演员名字，AI 组装 prompt

### 方案二：对比式生成（技术增强）

- 生成角色 B 时，将已确认角色 A 的图作为负参考传入
- 越往后生成的角色越不可能和前面的撞脸
- 取决于 Seedream 是否支持 negative image reference / IP-Adapter 反向权重

### 方案三：群像先行法（实验性）

- 先生成全阵容群像图，让模型在一次生成中被迫创造多样性
- 从群像中挑出合适的脸作为各角色视觉基准
- 好处是模型同画面内天然做差异化；坏处是可控性差，适合初期探索

### 方案四：人脸相似度自动卡控（兜底）

- 对所有角色正面照提取人脸 embedding
- 计算两两相似度矩阵，超阈值自动标红报警
- 可视化为散点图，一眼看出哪些角色在「脸空间」里扎堆
- 不解决问题但能自动发现问题，作为质量门禁

### 建议组合

**方案一（参考演员）+ 方案四（自动卡控）** 为主：
- 导演为每个角色选参考演员 → 成本低、效果强、天然差异化
- 生成完跑相似度检测 → 自动兜底
- 方案二（对比式生成）作为可选增强

## 影响

- `production/step3-bibles/` 的角色外貌章节需要支持导演编辑
- `production/step4-assets/` 确认后才批量生成
- 相关 skill 可能需要调整以支持这个人工介入环节
